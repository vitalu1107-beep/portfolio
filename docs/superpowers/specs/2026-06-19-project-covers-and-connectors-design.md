# 项目首图与动态连线设计说明

## 目标

在不改变首页项目卡片整体尺寸和交互方式的前提下，解决三个问题：

1. 四张项目首图与项目内容关联不足。
2. 项目标签没有清楚说明项目归属。
3. 画布连线使用固定坐标，卡片拖动后连线不能跟随。

## 首图方向

采用“业务模型信息图”，每张首图既是视觉封面，也是项目方法的第一层摘要。延续现有蓝、黄、绿、红项目色，但减少照片和PPT截图感，统一使用轻量网格、节点、箭头、数据标签与简洁图形。

### 独立项目｜小成就 APP

- 核心画面：手机产品界面与四步验证路径。
- 信息结构：洞察 → MVP → 上线 → 验证。
- 重点信号：AI辅助开发、PWA上线、第3天主动打开。
- 主色：绿色，辅以蓝色AI标记。

### 美团｜团买买

- 核心画面：供给增长飞轮。
- 信息结构：KOS → 商家 → 爆品 → GMV。
- 重点信号：520人KOS生态、单场GMV峰值120万。
- 主色：美团黄色，辅以绿色成交节点。

### 滴滴｜社群裂变

- 核心画面：同城私域增长漏斗。
- 信息结构：线下触达 → 社群沉淀 → 裂变传播 → 下单转化。
- 重点信号：100W+用户增量、单粉成本约1元。
- 主色：蓝色，辅以黄色触达节点。

### 滴滴｜618活动营销

- 核心画面：活动作战地图。
- 信息结构：目标 → 爆品 → 渠道 → 实时战报 → 风险预案。
- 重点信号：3天完成月度目标、GMV提升60%。
- 主色：橙红色，辅以黄色节奏节点。

## 卡片信息层级

项目卡片从上到下保持以下层级：

1. 项目归属标签。
2. 业务模型首图。
3. 结果型项目标题。
4. 一句话项目说明。
5. 两个关键成果。
6. 案例与外部链接。

归属标签固定为：

- 独立项目｜小成就 APP
- 美团｜团买买
- 滴滴｜社群裂变
- 滴滴｜618活动营销

## 动态连线

### 原因

现有连线是四个绝对定位的装饰元素，仅由CSS固定位置和角度。拖动卡片只更新卡片的 `transform`，连线无法获得新的端点坐标。

### 方案

- 用覆盖整个画布的SVG连接层替换固定CSS短线。
- 每条连线保存起点节点ID、终点节点ID和颜色。
- 根据节点初始布局坐标、节点尺寸和拖动偏移计算端点。
- 拖动过程中由现有 `positions` 状态驱动SVG路径重绘。
- 连接路径使用柔和曲线，避免穿过卡片主体。
- SVG层不接收鼠标事件，拖动、链接点击和画布移动不受影响。

### 坐标规则

- 连线坐标使用未缩放的画布坐标。
- 画布整体缩放时，卡片与SVG连接层一起缩放，不重复乘缩放比例。
- 卡片拖动位移已除以当前缩放比例，因此可直接与画布坐标相加。

## 组件边界

- `ProjectCover`：根据项目类型输出对应业务模型首图。
- `CanvasConnections`：接收连接关系和节点位移，输出SVG曲线。
- `ProjectNode`：继续负责项目卡片正文、成果和链接。
- `HomePage`：保存拖动位置，并把位置状态同时传给节点和连线。

## 异常与兼容

- 未识别项目类型时，首图回退为通用四步模型，不显示空白区域。
- 移动端现有规则隐藏自由画布连线，避免窄屏路径干扰内容。
- SVG连线使用 `pointer-events: none`，不会阻断点击。
- 项目数据缺少步骤或成果时，使用案例数据中的方法与指标回退。

## 验证标准

1. 四张首图在不读正文时也能区分项目类型。
2. 每张卡片都明确显示项目归属。
3. 拖动任意已连接卡片时，连线端点实时跟随。
4. 在10%、35%、72%、100%和130%缩放下，连线与卡片保持连接。
5. 首图在桌面和移动端无文字溢出、遮挡或空白。
6. 项目链接、画布拖动和卡片拖动继续正常工作。
